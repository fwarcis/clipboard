**Личный проект для возможности взаимодействия с системным буфером обмена внутри изолированных сред proot на Android.**

Сервер находится вне proot и использует CLI termux-api для обращения к системному буферу. Используется собственный протокол.

### Общая структура сообщения

1. Заголовок: код сообщения или команда
2. Разделитель заголовка и тела: ”\u001d” (Group Separator)
3. Тело (необязательно): возможная полезная нагрузка или метаинформация
4. Конец сообщения: “\u0000”

### Структура запроса клиента

1. Заголовок: команда ”COPY” или “PASTE”
2. Разделитель заголовка и тела
3. Тело (необязательно): текст для копирования или ничего
4. Конец сообщения

### Структура ответа сервера

1. Заголовок: код сообщения
2. Разделитель заголовка и тела
3. Тело: описание кода сообщения
4. Конец сообщения
